Photo Upload Without Draft - Implementation Summary

✅ Requirement Fulfilled: Users can now upload photos directly to Cloudinary without needing to save the listing as a draft first.

Backend Changes:

1. New Cloudinary Service Functions:
  • upload_temporary_photo() - Uploads photos to temporary folder
  • move_temporary_photos_to_listing() - Moves temp photos to permanent listing folder
  • cleanup_temporary_photos() - Cleans up old temporary photos
2. New API Endpoints:
  • POST /partner/temp-photos - Upload temporary photo
  • POST /partner/listings/{id}/move-temp-photos - Move temp photos to listing
  • DELETE /partner/temp-photos/{public_id} - Delete temporary photo
  • POST /partner/cleanup-temp-photos - Cleanup old temp photos

Frontend Changes:

3. New API Client Methods:
  • api.partner.uploadTempPhoto()
  • api.partner.moveTempPhotos()
  • api.partner.deleteTempPhoto()
  • api.partner.cleanupTempPhotos()
4. New React Hooks:
  • useUploadTempPhoto() - Upload photos to temporary storage
  • useMoveTempPhotos() - Move temp photos to permanent listing
  • useDeleteTempPhoto() - Delete temporary photos
  • useCleanupTempPhotos() - Cleanup old temporary photos
5. Updated OfferingsStep Component:
  • Removed draft requirement for photo uploads
  • Added temporary photo state management
  • Updated photo grid to show both permanent and temporary photos
  • Added visual distinction between saved and temporary photos
  • Updated photo count display

How It Works:

User Flow:
  1. User selects photos in listing builder (no draft save required)
  2. Photos are uploaded to temporary Cloudinary folder: temp/{partner_id}/{offering_type}/
  3. Photos are displayed with "Temporary" badge
  4. When listing is submitted/saved, temporary photos are moved to permanent location
  5. Old temporary photos are automatically cleaned up after 24 hours

Key Features:

  • No Draft Required: Photos can be uploaded immediately
  • Visual Feedback: Clear distinction between saved and temporary photos
  • Automatic Compression: Images are still compressed before upload
  • Cleanup System: Temporary photos are automatically cleaned up
  • Security: Partners can only access their own temporary photos
  • Organized Storage: Photos are organized by partner and offering type

File Structure:

Cloudinary Folder Structure:
  • temp/{partner_id}/{offering_type}/ - Temporary photos
  • listings/{listing_id}/{offering_type}/ - Permanent photos
✅ Result: Users can now upload photos immediately without any draft save requirement. Photos are stored temporarily and moved to permanent location when the listing is saved/submitted.
