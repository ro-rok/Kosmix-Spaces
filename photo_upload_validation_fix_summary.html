Photo Upload Validation Fix Summary

Original Error: Failed to upload Retail Product Pricelist L&T_page002_img_08.jpeg: Validation failed: undefined - undefined
✅ Root Cause Identified: The validation function had multiple issues:
  • Duplicate validate_image_file functions with different logic
  • ValidationError was being raised with None or undefined error messages
  • Missing proper error handling for edge cases

Fixes Applied:

1. Removed Duplicate Function:
  • Removed the second validate_image_file function
  • Kept the enhanced version with better validation logic
2. Enhanced Validation Function:
  • Added proper null/undefined checks for file, filename, and content_type
  • Added try-catch wrapper for unexpected errors
  • Improved error messages with specific details
  • Added validation for file existence
3. Fixed Error Handling in API Endpoints:
  • Added fallback error message: error_msg = error_message or "File validation failed"
  • Added proper ValidationError re-raising
  • Fixed all three photo upload endpoints

Updated Validation Logic:

New Validation Checks:
  1. File Existence: Checks if file object exists
  2. Filename: Ensures filename is provided
  3. Content Type: Validates content type exists and is image/*
  4. File Size: Max 15MB limit
  5. Format Support: JPEG, PNG, WebP, HEIC
  6. Security: Blocks dangerous file extensions
  7. Error Handling: Catches and reports validation exceptions

Supported File Types:

  • image/jpeg - JPEG images
  • image/jpg - JPG images
  • image/png - PNG images
  • image/webp - WebP images
  • image/heic - HEIC images (iPhone)

Error Messages Now Include:

  • "No file provided" - when file is null/undefined
  • "Filename is required" - when filename is missing
  • "File content type is missing" - when content type is null
  • "File must be an image" - when content type is not image/*
  • "File size must be less than 15MB" - when file is too large
  • "Allowed formats: JPEG, PNG, WebP, HEIC. Got: [type]" - when format is unsupported
  • "Invalid file type detected" - when dangerous extensions found
  • "File validation error: [details]" - for unexpected validation errors
✅ Result: Photo upload validation now provides clear, specific error messages instead of "undefined - undefined". The JPEG file mentioned in the error should now upload successfully or provide a clear reason why it failed.

Files Modified:

  • backend/app/services/cloudinary_service.py - Enhanced validation function
  • backend/app/routers/premium_listings.py - Fixed error handling in all photo upload endpoints
