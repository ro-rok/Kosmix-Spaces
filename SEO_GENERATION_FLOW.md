# SEO Generation Flow When Listing is Created

## Complete Flow Diagram

```
Partner Submits Listing
    ↓
POST /api/partner/listings/submit
    ↓
1. Create Listing Document (create_premium_listing)
   - Creates basic listing in MongoDB
   - Generates unique slug
   - Sets status: PENDING
   - NO SEO YET
    ↓
2. Add Hero Photos
   - Uploads hero photos to listing
    ↓
3. Add Offering Photos
   - Uploads photos for each enabled offering
    ↓
4. Generate SEO Metadata ⭐
   - Calls: update_listing_seo(listing_id)
   - Reads listing from MongoDB (now has photos)
   - Generates SEO from listing data
   - Saves SEO to MongoDB in seoMetadata field
    ↓
5. Submit for Review
   - Changes status to PENDING_REVIEW
    ↓
6. Return Response
   - Includes listing with SEO metadata
```

### Step 4: SEO Generation ⭐
**File**: `backend/app/routers/premium_listings.py` (lines 242-249)

```python
# Generate SEO metadata for the listing

    await update_listing_seo(listing_id)
```

**What `update_listing_seo()` does**:

1. **Fetches listing from MongoDB** (now has photos):
   ```python
   listing = await db.premium_listings.find_one({"_id": ObjectId(listing_id)})
   ```

2. **Calls `generate_listing_seo(listing)`** which:
   - Extracts: displayName, overview, locality, city, amenities, slug
   - Gets first hero photo for OG image
   - Generates meta title (50-70 chars optimal)
   - Generates meta description (150-160 chars optimal)
   - Extracts keywords from amenities, offerings, location
   - Creates canonical URL
   - Generates OG/Twitter metadata

3. **Saves to MongoDB**:
   ```python
   await db.premium_listings.update_one(
       {"_id": ObjectId(listing_id)},
       {
           "$set": {
               "seoMetadata": seo_metadata.model_dump(),  # ← Saved here!
               "updatedAt": datetime.utcnow()
           }
       }
   )
   ```

## SEO Generation Function Details

**File**: `backend/app/services/seo_service.py` (line 15-141)

### Input Data Used:
- `displayName` → Used in title
- `overview` → Used in description
- `locality`, `city` → Used in title, description, keywords
- `amenities` → Added as keywords
- `offerings` → Enabled offering types added as keywords
- `heroPhotos[0]` → Used as OG/Twitter image
- `slug` → Used for canonical URL

## Why SEO is Generated After Photos

SEO generation happens **after photos are added** because:
1. **OG Image**: Needs the first hero photo URL for social sharing
2. **Complete Data**: Listing has all information (photos, offerings) needed for accurate SEO
3. **Better Keywords**: Can extract keywords from enabled offerings

## When SEO is Regenerated

SEO is automatically regenerated in these scenarios:

1. ✅ **On Listing Submission** (initial creation)
2. ✅ **On Listing Approval** (when admin approves)
3. ✅ **On Listing Update** (when SEO-relevant fields change):
   - displayName
   - overview
   - locality
   - amenities
   - heroPhotos

## Frontend Usage

When frontend loads a listing page, it reads SEO from the stored `seoMetadata`:

```typescript
// PremiumSpaceDetail.tsx
const seoMetadata = listing.seoMetadata || {
  // Fallback if SEO not generated yet (shouldn't happen)
  metaTitle: `${listing.displayName} | ...`,
  // ...
}
```

This means SEO is:
- ✅ Generated once on submission
- ✅ Stored in MongoDB
- ✅ Reused on every page load (no regeneration needed)
- ✅ Updated automatically when listing content changes
